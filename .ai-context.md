## 📁 Version Information Locations

### Frontend Dependencies
- **File:** `frontend/package.json`
- **Lock file:** `frontend/bun.lockb`
- **Quick check:** `cd frontend && bun pm ls [package-name]`

### Backend Dependencies  
- **File:** `backend/Cargo.toml`
- **Lock file:** `backend/Cargo.lock`
- **Quick check:** `cd backend && cargo tree -p [crate-name]`

### Docker Images
- **File:** `docker-compose.yml`
- **Quick check:** `docker-compose config | grep image:`

### Consolidated Version Registry
- **File:** `./ESPERION_VERSIONS.md`
- **Contains:** All versions in one place with update history

---

## 🔍 Verification Commands

### Check Specific Package Version
```bash
# Frontend (Bun)
cd frontend && bun pm ls @nuxt/image

# Backend (Cargo)
cd backend && cargo tree -p surrealdb

# All frontend dependencies
cd frontend && bun pm ls --all
```

### Check Multiple Key Packages
```bash
# Frontend core
cd frontend && bun pm ls | grep -E "nuxt|vue|@nuxt"

# Backend core
cd backend && cargo tree -p axum -p tokio -p surrealdb | head -20
```

## ✅ AI Interaction Workflow

When user asks about versions:

1. **Acknowledge** the question
2. **Read ESPERION_VERSIONS.md** first
3. **Verify** with package manager commands if needed
4. **Answer** with current, verified information
5. **Cite** your sources ("According to package.json...")

### Example Interaction

**User:** "What version of @nuxt/image do we use?"

**Wrong ❌:** "You use @nuxt/image version 1.11.0"
*(This might be outdated)*

**Right ✅:** 
"Let me check the current version...

According to `frontend/package.json`, you're using:
- @nuxt/image: ^1.11.0

However, ESPERION_VERSIONS.md shows the latest available is 2.0.0 with breaking changes (xs/xxl screen sizes removed).

To verify the exact installed version:
```bash
cd frontend && bun pm ls @nuxt/image
```"
